### Accessible mode (body.accessible overrides)
- `--deep-blue: #1a1a2e`, `--teal: #0f4c75`, `--cream: #ffffff`, `--peach: #e8e8e8`, `--coral: #d4700a`

## Accessibility Image Swap
Images with `data-normal` and `data-accessible` attributes auto-swap when accessibility mode is toggled. All research figures have both normal and accessible versions.

## Figure Generation Conventions
- All figure scripts live in their respective project repos, output to `/home/james/JamesVonRaj.github.io/images/`
- Each script generates both normal (cream bg) and accessible (white bg) versions
- Shared geometry/data is generated once and passed to both versions (avoids mismatches from re-seeding)
- Font sizes: titles ≥16pt, axis labels ≥14pt, tick labels ≥12pt (readability is critical)
- Edge drawing order for conductance: sort descending (highest first), so low-conductance edges render on top
- Uniform edge widths in network plots (no variable thickness)
- Source nodes: coral squares. Target nodes: gold `#e8c547` triangles (large, with border outlines)
- BA networks use spring_layout (networkx) since they're not spatially embedded
- Physical learning detail figure: bottom row (loss curves) uses separate GridSpec spanning full width (not aligned with colorbar column)

## Writing Style
- No em dashes, en dashes, or `--`. Use commas, parentheses, or sentence breaks instead.
- Use "I" for solo work (SR metamaterials), "we" for collaborative work (TDA, physical learning)
- Don't oversell or overclaim. Descriptions should be honest and accurately scoped.
- Links in content sections (`.research-card-details`, `.about-text`, `section p`) are underlined with coral color
